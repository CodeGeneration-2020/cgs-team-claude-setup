# Data Model: Matchmaking

**Feature**: 009-matchmaking | **Date**: 2026-03-17

## Entities

### UserPreferences (existing — extended from 005-onboarding)

Add preference embedding for AI scoring.

| Field | Type | Description | Constraints |
|-------|------|-------------|-------------|
| preference_embedding | Vector | Embedding of concatenated preference text | Nullable, generated async on save/update |

**Notes:**
- Generated by embedding: `areas_of_interest + " " + geographic_preferences + " " + funding_focus`.
- Uses the same embedding model as project search (001-homepage).
- Regenerated when preferences are updated.
- Null if user has not completed onboarding.

---

## Derived Data (computed on-the-fly, no new table)

### MatchResult (response shape)

| Field | Type | Description |
|-------|------|-------------|
| projectId | String | Matched project ID |
| title | String | Project title |
| organizationName | String | Organization |
| thematicAreas | String[] | Project themes |
| location | String | Location |
| shortDescription | String | Brief description |
| fundingRequested | Decimal | Funding amount |
| status | String | Project status |
| tags | Object[] | Visual tags |
| compositeScore | Number | Overall match (0-1) |
| structuredScore | Number | Field matching score (0-1) |
| aiScore | Number | Vector similarity score (0-1) |
| matchReasons | String[] | Why matched (e.g., "Matches: Renewable Energy", "Region: East Africa") |

---

## No New Tables

Matchmaking scores are computed on-the-fly. Only change is adding `preference_embedding` vector column to the existing `user_preferences` table.
