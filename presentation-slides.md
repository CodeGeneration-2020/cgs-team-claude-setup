## SLIDE 8 — FIGMA ↔ CODE SYNC

_[Layout: heading top-left, four-step pipeline visualization]_

**Heading:** Figma Integration — Code ↔ Design Sync

**Capture** — AI captures every screen state from the live app → pushes to Figma at Desktop, Tablet, and Mobile viewports. Every state: empty, populated, loading, error, modal open.

**Link** — AI auto-matches Figma frames to specification screens. Each user story gets a direct design reference.

**Compare** — Pixel-level comparison with 2% threshold. Catches spacing, color, typography, layout differences automatically.

**Investigate** — AI Vision analyzes mismatches side-by-side: "Heading is 28px instead of 32px", "Card gap is 16px instead of 24px". Prioritized by severity: Critical, Major, Minor, Cosmetic.

## SLIDE 9 — 80% LESS MANUAL QA

_[Layout: heading top, three test types below]_

**Heading:** 80% reduction in manual QA

Our AI QA Agent generates three types of tests automatically:

**Unit Tests** — For all services, utilities, stores. 80%+ coverage enforced. Written before code (TDD).

**End-to-End Tests** — Generated from spec acceptance scenarios using Playwright. One test file per user story. Covers happy path, validation, empty states, error handling.

**Visual Comparison Tests** — Pixel-level: rendered UI vs. Figma design at every viewport. AI tells you not just how much differs, but exactly what and why.

**Result:** 90% fewer bugs in production. QA involved part-time for edge cases only.
